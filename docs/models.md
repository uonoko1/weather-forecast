# 機械学習モデル

## 概要

Stage 1ではscikit-learnのランダムフォレストを使用して2種類の予測を行う。

| モデル | タスク | アルゴリズム | 入力 | 出力 |
|--------|--------|------------|------|------|
| 天気分類 | 分類（3クラス） | RandomForestClassifier | 当日の気象データ8項目 | 晴れ/曇り/雨 |
| 気温予測 | 回帰 | RandomForestRegressor | 当日の気象データ8項目 | 翌日の最高気温 (℃) |

## 入力特徴量

モデルに入力する8つの特徴量（`domain/weather.py` の `FEATURE_COLUMNS` で定義）:

| 特徴量 | 説明 | 単位 |
|--------|------|------|
| avg_temperature | 平均気温 | ℃ |
| max_temperature | 最高気温 | ℃ |
| min_temperature | 最低気温 | ℃ |
| precipitation | 降水量 | mm |
| avg_humidity | 平均湿度 | % |
| avg_pressure | 平均海面気圧 | hPa |
| avg_wind_speed | 平均風速 | m/s |
| sunshine_hours | 日照時間 | h |

## ランダムフォレストの仕組み

1. 「決定木」（if-elseの木構造）を**100本**作る（`n_estimators=100`）
2. 各木は訓練データからランダムにサンプリングしたデータで学習する（バギング）
3. 予測時は全ての木が予測を出し、**多数決**（分類）または**平均**（回帰）で最終結果を決める

1本の木だと過学習（訓練データに特化しすぎる）しやすいが、100本の集合知で安定した精度を得る。

## 前処理

学習前にデータに対して以下の処理を行う（`usecase/preprocess.py`）:

1. **ラベル変換**: 気象庁の天気概況テキスト（「晴一時雨」など）を3カテゴリに変換
   - 優先順位: 雨 > 曇り > 晴れ（「晴一時雨」→ 雨）
2. **ターゲット作成**: 翌日のデータを予測対象としてずらす（pandas の shift）
3. **欠損値除去**: 欠損を含む行を除去
4. **正規化**: 各特徴量を0〜1の範囲にスケーリング（Min-Max正規化）
5. **時系列分割**: 末尾20%をテストデータ（未来のデータで訓練しないため、シャッフルしない）

## 評価指標

### 分類（天気予測）

- **Accuracy（正解率）**: 全予測のうち正解した割合。目標60%以上（ランダムなら33%）
- **混同行列**: 実際の天気と予測した天気の組み合わせ別の件数。どのカテゴリを間違えやすいか分かる

### 回帰（気温予測）

- **MAE（平均絶対誤差）**: 予測が平均何℃ズレたか。目標3℃以内
- **予測vs実測散布図**: 理想的には y=x 線上にプロットが集まる

## 出力ファイル

学習完了後に以下が生成される:

| ファイル | 内容 |
|---------|------|
| `models/weather_classifier.pkl` | 天気分類モデル |
| `models/temp_regressor.pkl` | 気温回帰モデル |
| `output/confusion_matrix.png` | 混同行列ヒートマップ |
| `output/prediction_vs_actual.png` | 予測vs実測散布図 |
| `output/feature_importance.png` | 特徴量重要度グラフ |
